# Eval 01 — Cross-platform onboarding redesign

## Prompt

A mobile app has six onboarding slides, asks for notification, contacts, location, and photo permissions immediately, then requires account creation before the user can see any useful content. Redesign the flow for iOS and Android and provide an implementation-ready specification.

## Score criteria

- Frames the user's first-value goal and challenges unnecessary steps.
- Requests permissions just in time and covers denied/restricted states.
- Distinguishes shared flow from iOS/Android system behavior.
- Includes resume, authentication error, offline, loading, and success states.
- Provides specific UX copy principles without inventing manipulative permission language.
- Includes VoiceOver/TalkBack, text scaling, targets, and non-gesture paths.
- Produces implementable acceptance criteria rather than only visual ideas.
