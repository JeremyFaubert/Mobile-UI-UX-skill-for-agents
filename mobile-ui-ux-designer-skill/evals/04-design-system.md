# Eval 04 — Mobile design system

## Prompt

A React Native app has 47 button variations, raw color values in screens, inconsistent loading states, and separate near-duplicate components for iOS and Android. Propose a design-system remediation plan and begin the implementation.

## Score criteria

- Inventories actual usage before creating a new abstract system.
- Defines primitive, semantic, component, and platform-resolved tokens.
- Consolidates by purpose and semantics rather than visual similarity alone.
- Preserves legitimate native differences.
- Includes states, text scaling, semantics, and target areas in component APIs.
- Proposes migration, deprecation, tests, and adoption metrics.
- Makes scoped code changes when repository context is available.
